# UML Диаграмма классов для проекта "Исторические события"

## Структура классов

### EventContainer (Класс)
**Атрибуты:**
- `_events: List[HistoricalEvent]` - список исторических событий

**Методы:**
- `__init__()` - конструктор, инициализация пустого контейнера
- `add(event: HistoricalEvent): void` - добавить событие в контейнер
- `remove(condition: Callable): int` - удалить события по условию, возвращает количество удаленных
- `print_all(): void` - вывести все события на экран

**Связь:** Агрегация с `HistoricalEvent` (контейнер хранит события)

---

### HistoricalEvent (Базовый класс)
**Атрибуты:**
- `name: str` - название события
- `date: str` - дата события

**Методы:**
- `__init__(name: str, date: str)` - конструктор
- `__str__(): str` - строковое представление события
- `__repr__(): str` - представление для отладки

**Связь:** Базовый класс для `Battle` и `Treaty` (наследование)

---

### Battle (Класс, наследуется от HistoricalEvent)
**Атрибуты:**
- `name: str` - название битвы (наследуется)
- `date: str` - дата битвы (наследуется)
- `place: str` - место битвы
- `type: str` - тип события ("Битва")

**Методы:**
- `__init__(name: str, date: str, place: str)` - конструктор
- `__str__(): str` - строковое представление битвы
- `__repr__(): str` - представление для отладки

**Связь:** Наследуется от `HistoricalEvent`

---

### Treaty (Класс, наследуется от HistoricalEvent)
**Атрибуты:**
- `name: str` - название договора (наследуется)
- `date: str` - дата договора (наследуется)
- `parties: str` - стороны договора
- `type: str` - тип события ("Договор")

**Методы:**
- `__init__(name: str, date: str, parties: str)` - конструктор
- `__str__(): str` - строковое представление договора
- `__repr__(): str` - представление для отладки

**Связь:** Наследуется от `HistoricalEvent`

---

## Типы связей

1. **Наследование (Generalization):**
   - `Battle` → `HistoricalEvent`
   - `Treaty` → `HistoricalEvent`

2. **Агрегация (Aggregation):**
   - `EventContainer` → `HistoricalEvent` (контейнер хранит события)

---

## Инструкция по использованию PlantUML

1. **В VS Code:**
   - Установите расширение "PlantUML"
   - Откройте файл `diagram.puml`
   - Нажмите `Alt+D` для предпросмотра диаграммы

2. **Онлайн:**
   - Перейдите на http://www.plantuml.com/plantuml/uml/
   - Скопируйте содержимое файла `diagram.puml`
   - Вставьте в редактор и получите диаграмму

3. **В Draw.io:**
   - Используйте описание выше для ручного создания диаграммы
   - Или экспортируйте из PlantUML в PNG/SVG и импортируйте в Draw.io
