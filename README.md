# Программа обработки исторических событий

## Описание

Программа обрабатывает файл с командами и выполняет операции над контейнером исторических событий.

## Структура проекта

- `historical_event.py` - классы для исторических событий (HistoricalEvent, Battle, Treaty)
- `container.py` - класс контейнера для хранения событий
- `command_parser.py` - парсер команд из файла
- `main.py` - главный файл программы
- `commands.txt` - пример файла с командами

## Типы исторических событий

### Битва
- Название (строка)
- Дата (строка)
- Место (строка)

### Договор
- Название (строка)
- Дата (строка)
- Стороны (строка)

## Команды

### ADD
Добавляет событие в контейнер.

**Формат:**
```
ADD <тип>|<название>|<дата>|<специфичный_параметр>
```

**Примеры:**
```
ADD Битва|Куликовская битва|1380|Куликово поле
ADD Договор|Версальский договор|1919|Германия и союзники
```

### REM
Удаляет события из контейнера по условию.

**Формат условий:**
- `type == "Битва"` или `type == "Договор"` - по типу события
- `name == "<название>"` - по точному названию
- `date == "<дата>"` - по дате
- `place == "<место>"` - по месту (только для битв)
- `parties == "<стороны>"` - по сторонам (только для договоров)
- `name contains "<подстрока>"` - по вхождению подстроки в название
- `date > "<дата>"` или `date < "<дата>"` - по сравнению дат

**Примеры:**
```
REM type == "Битва"
REM name == "Куликовская битва"
REM date > "1500"
REM name contains "война"
```

### PRINT
Выводит все события из контейнера на экран.

## Использование

```bash
python main.py commands.txt
```
