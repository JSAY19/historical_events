# Анализ решения проекта "Исторические события"

## Объяснение требований

### 1. Линтер (Linter)

**Линтер** — это инструмент, который анализирует исходный код на предмет:
- Стилистических ошибок (PEP 8 для Python)
- Потенциальных багов
- Плохих практик программирования
- Проблем с форматированием

**Популярные линтеры для Python:**
- **pylint** — комплексный анализ кода
- **flake8** — проверка стиля и ошибок
- **mypy** — статическая проверка типов

### 2. Статический анализатор

**Статический анализатор** проверяет код без его выполнения:
- Анализирует структуру кода
- Проверяет типы данных
- Находит потенциальные ошибки
- Выявляет проблемы безопасности

**Примеры:** pylint, mypy, bandit (безопасность)

### 3. Динамический анализатор / Профайлер

**Профайлер** анализирует программу во время выполнения:
- Измеряет время выполнения функций
- Анализирует использование памяти
- Находит узкие места (bottlenecks)
- Показывает, какие части кода выполняются чаще всего

**Примеры:** cProfile, line_profiler, memory_profiler

## Практическое применение

### Шаг 1: Установка инструментов

```bash
pip install -r requirements.txt
```

### Шаг 2: Анализ с помощью линтера (pylint)

```bash
pylint main.py historical_event.py container.py command_parser.py
```

### Шаг 3: Проверка стиля (flake8)

```bash
flake8 *.py
```

### Шаг 4: Проверка типов (mypy)

```bash
mypy *.py
```

### Шаг 5: Профилирование (cProfile)

```bash
python -m cProfile -s cumulative main.py commands.txt
```
